El proyecto se desarrolló utilizando Git y GitHub como sistema de control de versiones y colaboración entre equipos.
Primero, un integrante designado como líder creó el repositorio principal en GitHub, el cual funcionó como punto central para integrar todas las contribuciones. A partir de este repositorio, cada equipo realizó la clonación en su computadora local para contar con una copia completa del proyecto.
Posteriormente, cada equipo creó una rama independiente identificada con el nombre de su equipo. El uso de ramas permitió que los equipos trabajaran de forma paralela sin afectar el código ni los archivos del repositorio principal.
Dentro de su respectiva rama, los equipos desarrollaron su contribución asignada, ya fuera un reto simple de programación o la redacción de una parte del documento secuencial. Durante este proceso, cada integrante del equipo realizó al menos un commit, asegurando que todos participaran activamente y que los cambios quedaran correctamente registrados en el historial del proyecto.
Antes de subir los cambios al repositorio remoto, los equipos utilizaron los comandos git status para verificar el estado de los archivos y git log para revisar el historial de commits, confirmando que los cambios fueran correctos y completos.
Una vez verificada la rama, los equipos enviaron sus cambios a GitHub utilizando git push, y posteriormente crearon un Pull Request para solicitar la integración de su trabajo al repositorio principal. Cada Pull Request fue revisado por un equipo distinto, quien validó el contenido, realizó observaciones si fue necesario y finalmente aprobó la integración.
Finalmente, tras la aprobación, los cambios fueron fusionados al repositorio principal, logrando que el proyecto final integrara el trabajo de todos los equipos de manera ordenada, colaborativa y controlada.
